# CLAUDE.md

Guidance for working in this repo.

## What this is

A pure-frontend, Townscaper-style 3D building tool for students. The product is **a spatial-language editor**: the user defines *space* (rectangular volumes per floor, stacked) and the system **automatically materializes** the building skin (floors, exterior walls with deterministic windows + one ground door, corner columns, a continuous flat roof). The whole composition exports to a single **GLB**. Assets are Kenney's Building Kit (flat-roof only вҖ” no pitched roofs).

Stack: **React 19 + React Three Fiber v9 + @react-three/drei v10 + three 0.180 + Zustand 5 + Vite + TypeScript.**

## Run / build

```bash
node scripts/build-kit.mjs   # one-time: unpack the Kenney zip вҶ’ public/kit/* + pieces.json
npm install
npm run dev                  # http://localhost:5173
npm run build                # tsc -b && vite build  (always run before declaring done)
```

## Architecture (the important part)

**Truth вҶ’ derived.** The persisted truth is small; everything visible is derived.

- **Truth** (`store/useBuildStore.ts`, Zustand + persist): `cells` (`Record<"level,ci,cj", spaceId>`), `styleBySpace` (spaceIdвҶ’style), `faceOverrides` (`Record<"level,ci,cj,dir", 'wall'|'window'|'door'>`), `roofOverrides`, `stairs` (`Stair[]`). Plus session state (not persisted): `tool`, `activeLevel`, `selectedKeys` / `selectedStairId` / `selectedRoofId` / `selectedCols` (mutually-exclusive selections), `hoveredKey`, `hoverLevel` (surface-aware draw level under the cursor), `stairLanding` (stair-tool landing preview), `moveOverwrite` (еҲҶеүІгғ»з§»еӢ• overwrite toggle), undo/redo snapshots. `setTool` clears the transient hover/preview fields so they never bleed across tools.
- **Derive** (`kit/deriveSkin.ts`, **pure & deterministic**): `deriveSkin(cells, groundLvl, styleBySpace, faceOverrides, roofOverrides, fallback, stairs) вҶ’ Instance[]`. Same input вҶ’ same output (a stable `faceHash` chooses auto window/wall, no RNG/time). Re-runs on every edit inside `commit()`; the renderer/exporter just consume `instances`.
- **Render** (`three/`): `Scene` (Canvas, lights/sun, ground + grids, gizmo), `PlacedPieces`вҶ’`PieceInstance` (cloned GLB per instance via `useKitModel`), `GroundPlane` (pointerвҶ’cell, draw/erase + hover volume box), `exportGLB` (GLTFExporter, recenters X/Z).
- **Palette / recolour by SEMANTIC PART** (`kit/palette.ts` + `three/paletteBake.ts`): the kit shares ONE textured material `colormap` across all pieces (colour = UV into a flat-swatch palette atlas; no per-material colour, no AO) + a `glass` material on panes. Recolour is by **10 named parts** (`PaletteCat`): `wall / trim`(й—ЁзӘ—жЎҶ)`/ glass / door / door-metal`(й—Ёдә”йҮ‘)`/ roof / column`(жҹұиә«)`/ column-base`(жҹұзЎҖ)`/ floor / stair`. A vertex's part is `vertexCategory(pieceCategory(id), r,g,b)`: within a wall piece a near-white swatch (`isFrameColor`: lum>0.78 & sat<0.15) вҶ’ `trim`; within a column вҶ’ near-white = `column-base`, else `column`; within a door, the warm-red leaf (`r - max(g,b) > 20`) вҶ’ `door`, the cool/grey hardware (handle/kickplate/hinges) вҶ’ `door-metal`; floor/roof/stair are one colour each. The auto flat-roof's **top surface (`roof-flat-*`) follows the `wall` colour**, its **eave rail / cornice (`border`, gutters) follows the `column` colour** вҖ” neither is a "roof"; the **`roof` category colours only the DRAWN pitched/dome roofs** (`ProceduralRoofs`, `palette.roof`). `palette: Partial<Record<PaletteCat,string>>` (truth, persist v14, appearance-only вҖ” NOT in commit/undo/reskin); `colorFor(cat,palette)=palette[cat] ?? DEFAULT_COLOR[cat]` so the **default is a flat colour вүҲ the kit** (white frames/bases, blue-grey walls, dark roof вҖ” flat, no texture).
  - **Applied per-instance in `PieceInstance`** (`useLayoutEffect` on `[model, palette]`), NOT centrally: clones share their source GLB's geometry+materials by reference, so it (a) switches the colormap material to vertex-colour mode in place (`vertexColors=true; map=null`), (b) `bakeGeometry(geom, pieceCat)` once per geometry (sample colormap at each vertex UV + read its normal вҶ’ `vertexCategory` вҶ’ `userData.bakeCat`), (c) `recolorGeometry` вҖ” vertex = `colorFor(cat, palette)` (flat). A geometry `coloredFor` signature dedupes the recolour across instances; `glass` material is recoloured in place (restores its original when unset). **Pitfall: do NOT use `useGLTF(arrayOfUrls)` to mutate centrally вҖ” it returns different cached instances than the per-piece `useGLTF(url)`, so mutations don't reach the rendered clones.** Drawn procedural roofs follow `palette.roof ?? ROOF_COLOR`. Inside the export group вҶ’ the GLB carries the vertex colours (no texture). UI: рҹҺЁ й…ҚиүІ in the Toolbar вҶ’ `PalettePanel` (one colour picker per part + reset).

### Coordinate model (don't break this)
A **cell is the square between grid lines**: cell (i,j) spans `[GRID*i, GRID*(i+1)]`, center `GRID*i+HALF`. Walls/columns sit **on the lines**; floors fill the squares. `GRID=2.0`, `WALL_HEIGHT=2.4` (= fixed floor height; stacked floors must match the wall height). `toCell = Math.floor(v/GRID)`. Constants in `kit/constants.ts`, all measured from the GLBs.

### Skinning rules (deriveSkin)
- Floor per occupied cell. Exterior wall on any face whose neighbour cell is empty (occupancy only вҖ” connected cells merge into one open building, no interior walls).
- Window/wall is a deterministic hash per face; one auto **entrance door** on the ground floor's front (вҲ’Z) face (single + hinged leaf, offset by `DOOR_LEAF_OFFSET`).
- `faceOverrides` let the user force wall/window/door per face. **Contiguous same-kind window/door faces auto-merge into wide (2-cell) pieces** (`wall-*-wide-square`) вҖ” width emerges from how many adjacent faces share a kind.
- Roof = `roof-flat-center` per top cell + `border` rail along the outer edge (continuous flat roof). `roofFallbackCenter` drops the rail.
- **Drawn roofs (pitched/round)** вҖ” the kit has flat roofs only, so shaped roofs are **procedural geometry**, an additive layer over the flat roof. Truth: `roofs: RoofRegion[]` (`{level, ci0..cj1 rect, style: 'gable'|'hip'|'dome'|'shed', rotated?}`). In the **roof tool** (еұӢйЎ¶), drag a rect over a rooftop вҶ’ `addRoof` caps the building top inside the rect, trying the active level then the level just below (so drawing while standing on the rooftop **or** on the empty level above it both work), and stamps `activeRoofStyle`. Geometry is built in `three/roofGeometry.ts` (`roofGeometry(style,w,d,rise,rotated)` вҶ’ BufferGeometry, triangle-soup for crisp facets; **hip on a square footprint collapses to a pyramid ж”’е°–**; dome = scaled half-ellipsoid; ridge/slope runs the longer axis, `rotated` flips it) and rendered by `three/ProceduralRoofs.tsx` **inside the export group** (so GLTFExporter serializes it). The drawn roof **keeps the kit's flat roof + `border` eave (cornice)** untouched and the pitched cap **sits on top of the cornice** (`y0 + ROOF_CORNICE`), oversailing by `ROOF_EAVE` вҖ” so the kit's eave detail shows and there's no z-fight with the walls. Sized to the footprint вҶ’ 4Г—4 big, 3Г—3 small. **One roof per area**: `addRoof` replaces any existing roof it overlaps (same level, latest-wins). `commit` runs `pruneRoofs`, which keeps a roof only while its rect is a clean rooftop вҖ” building **on top of it** (a space drawn at `level+1`) or erasing its whole surface **invalidates and drops it** (draw a fresh one). deriveSkin is roof-agnostic (roofs are a pure render/export layer). Select a roof вҶ’ dock chips change style / rotate / delete; `Delete`/`Backspace` removes it. `selectedRoofId` is mutually exclusive with `selectedKeys`/`selectedStairId`.
- Per-**space** `styleBySpace` (spaceIdвҶ’style, not per-level) вҖ” a **structural/enclosure typology**, not a building program. Envelope is one of three kinds (`THEMES[style].walls` = `'full'|'low'|'none'`, ranked full>low>none): `enclosed` (full load-bearing walls + windows/doors; **corner** columns) / `semi` (post-and-slab **frame** + a waist-high perimeter parapet `wall-low`, balcony/loggia; **column grid**) / `open` (bare **frame**: no walls; **column grid**). An envelope piece is emitted on a face only when this cell's rank **exceeds** the neighbour's, and is this cell's kind (so enclosed-vs-semi вҶ’ one full wall from the enclosed side, no double). The column grid is at `BAY` spacing (world-aligned, incl. interior posts вҲӘ convex corners) for frame styles; `constants.ts`, default 2 cells/4m. One level can mix styles.
- **Circulation** (vertical, `kit/deriveCirculation.ts`) вҖ” full design rationale + the stair-model facts + pitfalls in **`docs/circulation-system.md`** (read before touching stair geometry). Summary: stairs are **derived, not hand-placed** вҖ” `deriveCirculation(cells, {groundDoors}, circulation) вҶ’ Stair[]` (deriveSkin computes the ground door faces). Two kinds of auto stair:
  - **Interior core** вҖ” per building (footprint-connected component), one vertical core, **solid model** (`stairs-closed`, set in `coreAt`), stacked per level (`coreAt` emits a flight only where a floor exists above вҶ’ only built where there's an upper storey). Placement obeys a **HARD stair invariant** (`flightUsable`): a flight needs an **approach cell at its foot** `(ciвҲ’di,cjвҲ’dj)` and a **step-off at its top** (a floor neighbour of B on `level+1`) вҖ” both open floor, so a stair **never starts at the wall root** (the recurring model-misread bug; foot/top geometry recorded in `docs/circulation-system.md` В§2). Among usable spots it's **scored, not door-anchored**: prefer the stair **tucked alongside a windowless wall, foot facing the room** (deriveSkin passes ground-floor opening faces `openFaces`). `seed`-chosen (re-roll = `seed++`). A footprint too small for *approach + 2-cell run* (e.g. 2Г—2) gets no auto core. (Manual stairs from `addStair`/rotate also default to `stairs-closed`.)
  - **Exterior stair = drawn PLATFORMS, chained by placement** (`expandPlatform(cells, platformKey, platformSet)`). In the stair tool you click empty outdoor cells вҶ’ added to `circulation.platforms` ("level,ci,cj"); each platform tile is a landing (and `addPlatform` opens a door on the adjacent enclosed wall). A platform emits **one flight toward the next platform** if one sits **one floor down + exactly 3 cells away in a cardinal direction** вҖ” the flight model fills 2 sloped cells (neither flat), so the next platform must be the flat landing *beyond* them: `platform(0) В· flight-high(1) В· flight-low(2) В· platform(3)`. **Direction follows the user's placement** вҖ” place the next platform LEFT vs RIGHT to fold back (switchback) vs go straight; the stair breaks at each platform (each is a landing). A platform with **no platform below** descends to the **nearest walkable surface вҖ” a building roof/terrace, or the ground** (whichever the run reaches first; `descendFlights`). **Building roofs are platforms too**, so a setback's roof is a valid landing and the stair stops on the terrace instead of being forced to the ground; auto-direction **prefers a terrace landing**, then the shortest run (`pickDescent`). All flights stay OUTSIDE the footprint (never cross a wall) вҖ” the only cell a flight may overlap is the roof it lands on. **Per-platform overrides** (select the tower in the select tool): `circulation.platformModel[key]` swaps the stair model (`stairs-open`/`-center`/`-closed`, all 2-cell drop-ins; `stairs-sides` is 3-cell, deferred), and `circulation.platformDir[key]` ("rotate") forces which edge a *straight* descent goes down (ignored if that direction isn't clear; chained towers ignore it вҖ” placement decides). **Hard invariant: every flight stays OUTSIDE the footprint вҖ” never crosses a wall** (both tangents blocked вҶ’ platform only, no stair). No central well, no floating mid-air platforms вҖ” the only platform is the one you drew at the door. `addPlatform` also opens a `door` on the adjacent enclosed wall face (`platformDoorFace`). Flights share `stairKey = platform:<key>` (delete removes the group). (Stair model `stairs-open` measured from the GLB: pure diagonal, full-height step at the +Z edge, no built-in landings.)
  - **Interior vs exterior is by source**: interior auto-cores (`auto:*`/manual) cut `floorHoles` on `level+1`; **`platform:*` stairs never cut a hole** вҖ” even when a roof landing puts their bottom cell inside the footprint they sit ON the terrace, not through it (deriveSkin's hole loop skips `platform:` ids).
  - Overrides: `circulation.manual[]` (user-drawn stairs, kept) + `circulation.suppressed[]` (deleted auto cores, by `auto:вҖҰ` id) + `circulation.platforms[]` (drawn exterior landings, by `"level,ci,cj"`; delete a flight removes the whole group via its `platform:<key>` stairKey); `circulation.auto` toggles the interior auto layer. A `Stair = {level, ci, cj, dir}` is one flight (`stairs-open`, 2 cells deep, high end faces `dir`). Additive вҖ” never changes wall/column/roof/style logic.

### Interaction (fixed Tinkercad convention вҖ” never swaps)
Right-drag = orbit, middle-drag = pan, wheel = zoom (all tools). Left = the active tool: **select** (default; click a wall face *or a stair* to select вҖ” faces: Shift to multi-select + arrow keys cycle the style; a selected stair shows rotate/delete chips, `Delete`/`Backspace` removes it) / **space** (drag a rect to fill cells вҖ” **surface-aware level**: see below) / **move** (еҲҶеүІгғ»з§»еӢ•: drag a whole building to reposition it, OR marquee a rectangle of columns and drag the 3D handle to move *part* of a building вҖ” moving half away auto-splits it into two walled buildings вҖ” see below) / **stair** (circulation: auto-places interior stair cores at doors вҖ” dock has иҮӘеҠЁ toggle + жҚўдёҖдёӘ/re-roll; **click a wall face OR any walkable surface edge** (floor/rooftop, incl. open/semi frames) to snap a landing platform just outside it at the right level, or click an empty outdoor cell to drop one at `activeLevel` вҖ” either auto-grows a stair down) / **roof** (еұӢйЎ¶: drag a rect over a rooftop вҶ’ a procedural gable/hip/dome/shed roof, **surface-aware** so no floor switch; style chips in the dock; select a roof to restyle/rotate/delete) / **erase** (area eraser, **surface-aware, the mirror of draw**: point at a rooftop вҶ’ peel the floor you see the top of вҖ” no floor-switch вҖ” wiping everything anchored in the dragged rect on that level: cells, manual stairs, **exterior platform stairs** (landing cell in rect), and **drawn roofs** (rect overlaps the roof at this level, or the level below when erasing the empty floor above it вҖ” mirrors drawing a roof from above); auto interior stairs re-derive). Selection state: `selectedKeys` (faces), `selectedStairId` (one stair), and `selectedRoofId` (one roof) are mutually exclusive. `OrbitControls keyEvents={false}` so arrows don't pan. The blue per-level **drawing grid** (floating at the active level) shows **only under a drawing tool** вҖ” it's hidden in select **and move** mode (they clutter picking; selection/move are per-object, not per-level); the faint ground baseline grid stays. **Surface-aware level** (space, roof, erase вҖ” all rect tools): the touched surface under the cursor = the base of a reference level **R** (`resolveDrawTarget`, `kit/pickLevel.ts`, pure вҖ” scans horizontal planes top-down for a rooftop вҶ’ `rooftop+1`, else `activeLevel`). **Draw and erase are mirror ops across that plane**: **space** fills R / **roof** caps it (`addRoof`'s `topBox(R) ?? topBox(R-1)`); **erase** wipes **R-1** вҖ” the floor whose *top* is the touched surface (the one you see) вҖ” so no manual floor-hunt in tall buildings. All three drag on the **same R plane** (the visible touched surface) and show their preview there (space/roof = a volume box above the roof; erase = a flat red marker *on* the roof, shown only where a floor exists to peel). `fillSpace`/`addRoof` act on R, `eraseCells` on R-1; each takes the level explicitly. Point at empty ground вҶ’ R = `activeLevel` (draw there; erase R-1 is a safe no-op). The resolve is **transient** вҖ” it never changes `activeLevel` (manual `FloorStrip`/arrows still work). `hoverLevel` (= R) drives the floating grid + the `FloorStrip` **target** highlight (green ring, vs the blue solid active-floor highlight) вҖ” identical feedback for draw and erase, only the action differs. **Stair tool snapping** (pick a target in the stair tool вҶ’ a landing platform snaps just outside it, at the right level, no floor-switch):
  - **Wall-snap** (enclosed): an exterior wall face is pickable (`PieceInstance`) вҖ” clicking drops a landing in the outdoor cell outside that face, at the face's level (`addPlatformAtFace(faceKey)` вҶ’ outdoor О” per dir W0/E1/S2/N3 вҶ’ `addPlatform(pi,pj,level)`). Hovered face highlights (`SelectionOverlay` shown in the stair tool too).
  - **Surface-edge snap** (any style incl. `open`/`semi` frames + **rooftops**): flat-roof pieces (always) and **open/semi floor** pieces carry a `surfaceKey = "landLevel,cellLevel,ci,cj"` (floor: land=cell=L; roof: land=L+1 since you stand on top). **Enclosed floors are NOT tagged** вҖ” you reach them via their wall (wall-snap); making them pickable let an aimed-at wall mis-hit the floor and scatter platforms on the ground. In the stair tool they're pickable; `surfaceEdgeLanding(cells, surfaceKey, hitX, hitZ)` (`kit/pickLevel.ts`, pure) picks the **nearest exterior edge** of the pointed cell from the hit point and returns the empty outdoor neighbour at `landLevel` вҶ’ `addPlatformAtSurfaceEdge` places it there (no door for open/semi вҖ” no wall). Pointing at a fully-interior cell returns null (**no stair in the middle of a surface** вҖ” only edges). Hover previews the landing (`stairLanding` session state вҶ’ `GroundPlane`).
  - Clicking empty ground still drops a platform at `activeLevel` (for chained/switchback descents). Shortcuts: V/S/M/T/E tools; **вҶ‘/вҶ“ = floor up/down** (`stepLevel`: вҶ‘ can reach one floor above the topmost built level, then stops; empty scene stays at 0); **вҶҗ/вҶ’ = cycle the selected face's style**; `[`/`]` also change level.

**Move / split a building** (`move` tool, labelled **еҲҶеүІгғ»з§»еӢ•**; `GroundPlane` + `MovePreview` + `SpaceFieldOverlay` + `MoveGizmo` + store `moveBuilding`/`selectedCols`): the truth is cell-keyed, so moving = translating those keys by a grid `(О”i,О”j)`, and **splitting/healing is free** вҖ” `deriveSkin` is occupancy-based, so moving a *subset* of a building's columns away re-derives walls on the newly-exposed cut edge (one building becomes two, each fully walled); moving a subset next to another building merges (shared edge вҶ’ no wall). Two ways to grab:
  - **Press on a building + drag** вҶ’ the **whole connected footprint** (`footprintAt` = BFS over columns occupied on *any* level, 4-connected), green/red ghost, release вҶ’ `moveBuilding`. Fast, one gesture.
  - **Press on empty ground + drag a rectangle** вҶ’ marquee-select the occupied columns inside it into `selectedCols`; the selection (and the drag rect) is shown as a **`SpaceFieldOverlay`** вҖ” a зөҗз•Ң-style light volume rising from the ground and fading to the sky (animated scan band), conveying "a column of space, ground to sky" (not just the active floor). A 3D **`MoveGizmo`** knob floats above the selection centre вҖ” drag it to slide the selection (green/amber/red ghost), or use **arrow keys** to nudge 1 cell (Up = вҲ’Z). Release вҶ’ `moveBuilding(selectedCols, вҖҰ)`; the selection follows to the new spot. Click empty / Esc clears.
  
  `moveBuilding(cols, di, dj)` translates **everything anchored to those columns** in one `commit` (so undo/redo + pruneRoofs + reskin all run): `cells` (spaceId kept вҶ’ `styleBySpace` untouched), `faceOverrides`, drawn `roofs` (**split by cell**: each roof's cells are partitioned into moved vs stayed and each side re-fit to a bounding rect вҖ” so cutting a building in half yields a roof over each half, styles preserved; `pruneRoofs` drops any leftover. Merge is not auto вҖ” adjacent halves stay separate, redraw for one big roof. bbox over-covers non-rectangular subsets, tolerated by `pruneRoofs`), `circulation.manual` / `suppressed` (parse the `auto:вҖҰ` id) / `platforms` + `platformModel` + `platformDir` (the **whole exterior-stair chain** follows: seed = platforms in/4-adjacent to the moved columns, then BFS along the chain link вҖ” a platform one level up/down and exactly 3 cells away in a cardinal вҖ” so far/floating landings come too; a chain cut by a partial marquee just splits into two towers, re-derived safely). Auto interior cores aren't stored вҶ’ they re-derive at the new spot. **Overlap handling** depends on the `moveOverwrite` toggle (гҖҢдёҠжӣёгҒҚгҖҚchip in the move dock, default off): the check runs **up-front** so it can decide before committing. **Off (default) = block**: if any target cell lands on an occupied cell of a *non-moving* column the move is a true no-op (no undo snapshot), ghost is **red**. **On = overwrite**: the mover wins вҖ” its cells (and thus `spaceId` вҶ’ style) overwrite the target's at the overlapping keys (ghost **amber**), so e.g. an `enclosed` block buried into an `open` one turns that region enclosed; `cells` is built non-movers-first-then-movers so movers overwrite, stale `faceOverrides` on overwritten cells are dropped, and `pruneStyles` removes any space left with zero cells (fully buried). Partial overlap вҶ’ the two merge into one connected building with mixed per-space styles (allowed). Undo reverts it. The three ghost states are green (`ok`) / amber (`overwrite`) / red (`blocked`) via `MovePreview`'s `mode`. `cols` is any set of `"i,j"` columns (whole footprint or a marquee subset). Appearance-only `palette` is not touched.

**Exterior stairs** are drawn, not auto: in the **stair tool**, go to the floor with the door, click the empty outdoor cell just outside it вҶ’ that drops a **landing platform**, and a wall-parallel stair auto-descends from it to the nearest walkable surface вҖ” a building roof/terrace or the ground (see Skinning rules вҶ’ Circulation). You control the platform position exactly; the stair falls out of it.

## Verification
**Logic tests are the default** (bundle a `kit/*.ts` with `npx esbuild --bundle` and assert piece counts/positions in node) вҖ” cheap and deterministic. **Screenshots are the single most expensive step (image tokens); take at most one, and only when geometry/orientation genuinely can't be confirmed from logic.** A headless harness exists (Playwright + chromium swiftshader, store exposed on `window.useBuildStore` in dev). See `[[reference-playwright-harness]]` memory. Always `npm run build` before finishing.

**Working cadence (cost):** do **one feature per session** вҖ” context grows fast and every turn re-reads it, so a fresh session per feature keeps each turn cheap (lean on this CLAUDE.md + memory to get back up to speed). See `[[feedback-verification-cost]]`.

## Gotchas
- three 0.180 OrbitControls dolly methods are private (`_dollyIn`); the nav zoom in `Scene`'s `CameraBridge` scales `camera.position` instead.
- GLTFExporter doesn't keep instancing; fine at school scale.
- Don't reintroduce a hand/`grab` cursor (misleading вҖ” camera isn't on left).
- The auto **flat** roof has no per-cell style switch (kit ships flat tiles only). Shaped roofs (gable/hip/dome/shed) are a separate **drawn, procedural** layer вҖ” see the drawn-roofs bullet under Skinning rules; they are generated three.js geometry, not GLB pieces.
- **R3F event passthrough**: `GroundPlane` places on `onPointerDown`/`onPointerUp`, but `PieceInstance` picks on `onClick`. A piece that lacks an `onPointerDown` handler does NOT consume the press вҖ” R3F forwards `pointerdown` to the next hit object *behind* it (the giant `GroundPlane`), so clicking a piece can ALSO fire the ground handler. `onClick`'s `stopPropagation` can't stop `pointerdown`/`up`. Pickable pieces therefore include `onPointerDown: (e) => { if (e.button===0) e.stopPropagation(); }` вҖ” without it, a stair wall/surface click also dropped a second platform on the ground behind the wall. Any new mesh that participates in left-click placement must consume the press the same way.
